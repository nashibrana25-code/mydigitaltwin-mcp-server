## üìä Comprehensive Technical Interview Prep by Domain

### SQL Interview Questions (Jobs #2, #3, #5, #9)

**Q1: "Write a SQL query to find employees earning more than their manager."**
```sql
SELECT e.name, e.salary
FROM employees e
JOIN employees m ON e.manager_id = m.employee_id
WHERE e.salary > m.salary;
```

**Q2: "Explain the difference between INNER JOIN and LEFT JOIN."**
- INNER JOIN: Returns only matching rows from both tables
- LEFT JOIN: Returns all rows from left table + matching rows from right (NULL if no match)

**Example from your work:**
"In my Library Management System project, I used LEFT JOIN to show all books even if they had no current borrowers, and INNER JOIN to show only books that were currently borrowed."

---

**Q3: "How would you find duplicate records in a table?"**
```sql
SELECT column_name, COUNT(*)
FROM table_name
GROUP BY column_name
HAVING COUNT(*) > 1;
```

"I used this pattern in my internship to identify duplicate data chunks when processing my profile for embeddings."

---

**Q4: "Explain database normalization."**
"Normalization is organizing data to reduce redundancy:
- 1NF: Eliminate repeating groups, atomic values
- 2NF: Remove partial dependencies
- 3NF: Remove transitive dependencies

Example: In Library Management System, I separated books, members, and transactions into different tables rather than storing all data in one table, reducing redundancy and improving data integrity."

---

**Q5: "How do you optimize a slow SQL query?"**
1. Check query execution plan (EXPLAIN)
2. Add indexes on frequently queried columns
3. Avoid SELECT *, retrieve only needed columns
4. Use proper JOIN types
5. Consider query restructuring or denormalization for read-heavy tables

"In my internship, I optimized vector database queries through indexing and caching, achieving sub-second response times."

**Q: "Reverse a string in Python."**
```python
def reverse_string(s):
    return s[::-1]

# Alternative
def reverse_string_iterative(s):
    result = ""
    for char in s:
        result = char + result
    return result
```

**Q: "Find duplicates in a list."**
```python
def find_duplicates(lst):
    seen = set()
    duplicates = set()
    for item in lst:
        if item in seen:
            duplicates.add(item)
        seen.add(item)
    return list(duplicates)
```

**Q: "Explain financial variance analysis."**

**Variance Analysis Framework:**
1. **Calculate Variance:** Actual - Budget (or Forecast)
2. **Express as %:** (Variance / Budget) √ó 100
3. **Categorize:** Favorable (positive for revenue/negative for costs) vs. Unfavorable
4. **Investigate Root Causes:**
   - Price variance: Due to price changes
   - Volume variance: Due to quantity changes
   - Mix variance: Due to product/service mix
5. **Recommend Actions:** Corrective measures or process improvements

"While I haven't done corporate variance analysis, I performed technical variance analysis in my internship‚Äîcomparing actual API costs vs. projected costs and investigating why caching reduced costs by 60% instead of projected 40%, identifying query pattern changes as the reason."

---

**Q: "How would you build a financial model?"**

**Financial Modeling Steps:**
1. **Define Objective:** What decision needs to be made?
2. **Identify Inputs:** Revenue drivers, cost assumptions, growth rates
3. **Build Calculations:** Formulas for revenue, costs, EBITDA, cash flow
4. **Add Scenarios:** Best case / base case / worst case
5. **Sensitivity Analysis:** How changes in inputs affect outputs
6. **Validate:** Check formulas, ensure logic is sound
7. **Document Assumptions:** Clear notes for all inputs

"In my internship, I built a cost model for my digital twin system with inputs (API call frequency, cache hit rate, service costs) and calculated scenarios showing 40-80% cost reduction range based on caching effectiveness. This is the same approach I'd use for Samsung's financial models."
